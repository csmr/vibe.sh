Code Guidelines:

    Functional programming code, mix-ins, modules.
    Don't repeat yourself (DRY).
    Expressions formatted one thing per line.
    One thing functions, limit blocks to >11 lines.
    Lines as short as possible (<80 chars).
    Expressing the essential in as few words as possible.


Approach

    Think before acting. Read existing files before writing code.
    Be concise in output but thorough in reasoning.
    Prefer editing over rewriting whole files.
    Do not re-read files you have already read unless the file may have changed.
    Test your code before declaring done.
    No sycophantic openers or closing fluff.
    Keep solutions simple and direct. No over-engineering.
    If unsure: say so. Never guess or invent file paths.
    User instructions always override this file.
    Expressing the essential in as few words as possible.


Efficiency

    Read before writing. Understand the problem before coding.
    No redundant file reads. Read each file once.
    One focused coding pass. Avoid write-delete-rewrite cycles.
    Test once, fix if needed, verify once. No unnecessary iterations.
    Budget: 50 tool calls maximum. Work efficiently.
    Expressing the essential in as few words as possible.
